# ADR-001: MCP Roots не интегрируются с агентом

## Дата
2026-06-06

## Статус
Accepted

## Контекст

В проекте реализована поддержка MCP Roots согласно MCP спецификации. Возник вопрос: нужно ли автоматически устанавливать roots из session.cwd и интегрировать их с логикой агента?

### Архитектура коммуникации

```
┌─────────────┐         ACP          ┌─────────────┐         MCP          ┌─────────────┐
│             │  fs/read_text_file   │             │  tools/call          │             │
│     IDE     │ ───────────────────> │   codelab   │ ───────────────────> │  MCP Server │
│  (клиент)   │ <─────────────────── │   (агент)   │ <─────────────────── │  (external) │
│             │    file content      │             │    tool result       │             │
└─────────────┘                      └─────────────┘                      └─────────────┘
                                              │
                                              │ MCP Roots
                                              │ (file:///project)
                                              ▼
                                       MCP сервер узнаёт
                                       о границах проекта
```

### Два разных направления

| Протокол | Направление | Назначение |
|----------|-------------|------------|
| **ACP** | Агент → IDE | Агент **запрашивает** у IDE чтение/запись файлов |
| **MCP Roots** | Клиент → MCP серверы | Клиент **сообщает** MCP серверам о границах проекта |

## Решение

**НЕ реализовывать автоматическую интеграцию MCP Roots с агентом** по следующим причинам:

### 1. Избыточность

Агент уже имеет доступ к файлам через ACP:
- `fs/read_text_file` — чтение файлов
- `fs/write_text_file` — запись файлов
- IDE выполняет эти запросы

MCP filesystem server становится избыточным — зачем подключать его, если агент уже может читать файлы через ACP?

### 2. Архитектурная чистота

Не нужно смешивать ACP и MCP для работы с файлами. Каждый протокол имеет свою зону ответственности:
- ACP — взаимодействие агента с IDE
- MCP — взаимодействие агента с внешними инструментами

### 3. Практическая польза

MCP Roots полезны только для специфичных MCP серверов:
- MCP серверы, которые работают с файлами (git, database schema)
- Когда нужно сообщить MCP серверу о "текущем проекте"
- Для совместимости с MCP спецификацией

MCP Roots избыточны для:
- MCP filesystem server (агент уже имеет доступ через ACP)
- MCP серверов, которые не работают с файловой системой
- MCP серверов, которые игнорируют roots

## Что реализовано

✅ Поддерживать MCP Roots как часть MCP спецификации:
- Модели `MCPRoot`, `MCPClientCapabilities`
- Обработка `roots/list` от серверов
- Отправка `notifications/roots/list_changed`
- Возможность ручной настройки roots в конфигурации

## Что НЕ реализовано (и не будет)

❌ Автоматическая установка roots из `session.cwd`
❌ Интеграция с tool execution
❌ Валидация путей в tools
❌ Автоматическое обновление roots при смене cwd

## Когда пользователь может использовать MCP Roots

Пользователь может явно настроить roots в конфигурации MCP серверов, если:
- MCP сервер поддерживает roots и использует их для работы
- Нужно сообщить MCP серверу о границах проекта
- MCP сервер не является filesystem server (для filesystem server используется ACP)

### Пример конфигурации

```json
{
  "mcpServers": {
    "git-server": {
      "command": "mcp-git-server",
      "roots": [
        {"uri": "file:///home/user/project", "name": "Project Root"}
      ]
    }
  }
}
```

## Последствия

### Положительные
- ✅ Избегаем избыточности в архитектуре
- ✅ Сохраняем чистоту разделения ответственности между ACP и MCP
- ✅ Упрощаем код (не нужна автоматическая синхронизация roots)
- ✅ Соответствуем MCP спецификации (roots поддерживаются, но не навязываются)

### Отрицательные
- ⚠️ MCP filesystem server может быть подключён, но будет избыточным
- ⚠️ Пользователь должен понимать разницу между ACP и MCP для работы с файлами

## Альтернативы

### Альтернатива 1: Автоматическая установка roots из session.cwd
**Отклонена** — создаёт избыточность с ACP

### Альтернатива 2: Интеграция roots с tool execution
**Отклонена** — смешивает ответственность ACP и MCP

### Альтернатива 3: Полная поддержка roots (реализовано)
**Принята** — поддерживаем roots как часть MCP спецификации, но не интегрируем с агентом

## Вывод

В архитектуре codelab, где агент взаимодействует с IDE через ACP, MCP Roots не нужны для интеграции с агентом. Они поддерживаются как часть MCP спецификации для совместимости с MCP серверами, которые их используют, но автоматическая интеграция с агентом не реализована и не будет реализована из-за избыточности с ACP.
