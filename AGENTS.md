# AGENTS

Инструкции для агентных ассистентов в этом репозитории.

## Контекст проекта

* Единый Python-проект, объединяющий сервер и клиент ACP.
* Менеджер окружения и запуск команд: `uv`.
* Базовые проверки запускаются через `Makefile` из корня репозитория.
* Основной язык документации: русский.
* Минимальная поддерживаемая версия Python: 3.12.

---

## Запрещено

### Протоколы

* Нарушать ACP, A2A или MCP спецификации.
* Менять внешние протоколы без отдельного архитектурного решения.
* Изменять документацию в `doc/Agent Client Protocol/`, `doc/Agent To Agent Protocol/`, `doc/Model Context Protocol/`.

* Менять публичные контракты без проверки совместимости.

* Добавлять зависимости без необходимости.
* Обходить архитектурные ограничения проекта.

* Удалять функциональность без явного запроса.

## Приоритет правил

При конфликте требований:

1. Корректность работы системы.
2. Соответствие ACP, A2A и MCP спецификациям.
3. Сохранение обратной совместимости.
4. Минимальность изменений.

## Правила кода

### Минимальность

Один логический блок = одна задача. Предпочитать расширение существующих компонентов созданию новых. Не перемещать файлы/классы без необходимости.

### Абстракции

Перед созданием нового файла, класса, сервиса, менеджера — проверить существующие реализации и возможность расширения. Новая абстракция — исключение, а не правило.

### Повторное использование

Переиспользовать компоненты, избегать дублирования, предпочитать композицию копированию.

### Типизация

Python 3.12+. Не ослаблять типы. Не использовать `Any` без необходимости. Не использовать `# type: ignore` без обоснования.

### Ошибки

Не скрывать. Сохранять причину через `raise ... from ...`. Использовать существующую инфраструктуру логирования. Новые точки отказа — с понятными сообщениями.

### Async

Не блокировать event loop. Не использовать sync I/O в async-коде. Корректно обрабатывать `asyncio.CancelledError`. Не создавать фоновые задачи без контроля жизненного цикла.

### Комментарии

Только там, где логика неочевидна. Запрещены комментарии, пересказывающие код. Публичные API — через docstring. Не оставлять закомментированный код.

Плохо:
```python
# Добавляем сообщение в список
messages.append(message)
```

Хорошо:
```python
# ACP требует сохранять исходный порядок сообщений для корректного воспроизведения истории.
messages.append(message)
```

## Протоколы и совместимость

**Обратная совместимость:** по умолчанию. Изменение публичного поведения = потенциально несовместимое изменение.

**Изменения форматов** (сериализация, хранение, конфигурация, сессии, схемы данных) — с миграцией или механизмом обратной совместимости.

**Удаление кода** допустимо только если: функциональность не используется, нет влияния на публичные API, протоколы, обратную совместимость.

**Изменения протокола** (требуют отдельной проверки): JSON-RPC сообщения, ACP Content Types, Session State, Tool Call State, публичные ACP/MCP API, форматы сериализации, transport layer контракты.

## Архитектура

**Клиент** — Clean Architecture. Зависимости: `domain ← application ← infrastructure ← presentation ← tui`. Внутренние слои не зависят от внешних.

Запрещены зависимости: `domain → application/infrastructure/presentation/tui`, `application → infrastructure/presentation/tui`.

**Сервер:** не обходить ACP Protocol Layer. Не обращаться напрямую к состоянию сессий при наличии менеджера. Использовать существующие Registry, Factory, Manager. Соблюдать границы protocol, storage, agent, tools, llm.

## Технические решения

### MCP SSE Transport

`SseTransport` помечен как deprecated в MCP specification. Удалять запрещено: используется существующими клиентами, необходим для обратной совместимости. Любые изменения должны сохранять поддержку текущего поведения.
