# Spec: Model Discovery

## ADDED Requirements

### Requirement: Интерфейс обнаружения моделей
Система ДОЛЖНА определять абстрактный базовый класс `ModelDiscovery`, который реализуют все реализации обнаружения моделей.

#### Scenario: Обнаружение моделей
- **WHEN** вызван `ModelDiscovery.discover_models(config)`
- **THEN** возвращён список `ModelInfo` для доступных моделей

### Requirement: Статическое обнаружение (по умолчанию)
Система ДОЛЖНА предоставлять реализацию `StaticDiscovery`, возвращающую предконфигурированный список моделей.

#### Scenario: Возврат статических моделей
- **WHEN** `StaticDiscovery` инициализирован списком `ModelInfo`
- **THEN** `discover_models()` возвращает этот список

#### Scenario: Пустой список моделей
- **WHEN** `StaticDiscovery` инициализирован пустым списком
- **THEN** `discover_models()` возвращает пустой список

### Requirement: Регистрация моделей провайдера
Система ДОЛЖНА позволять провайдерам регистрировать доступные модели при инициализации.

#### Scenario: Провайдер регистрирует модели
- **WHEN** провайдер инициализирован
- **THEN** его модели зарегистрированы в `LLMProviderRegistry`

#### Scenario: Модели отображаются в configOptions
- **WHEN** модели зарегистрированы для провайдера
- **THEN** они отображаются в списке опций configOption `model`

### Requirement: Динамическое обнаружение (Extension Point)
Система ДОЛЖНА поддерживать динамическое обнаружение моделей для локальных провайдеров (Ollama, LMStudio) как будущее расширение.

#### Scenario: Ollama обнаруживает модели через API
- **WHEN** вызван `OllamaDiscovery.discover_models(config)` и Ollama запущен
- **THEN** выполнен запрос `GET /v1/models` и возвращены обнаруженные модели

#### Scenario: Ollama не запущен
- **WHEN** вызван `OllamaDiscovery.discover_models(config)` и Ollama не запущен
- **THEN** возвращён список моделей по умолчанию (fallback)

#### Scenario: Фоновое обновление
- **WHEN** провайдер поддерживает фоновое обновление моделей
- **THEN** отправлено событие `ModelsUpdated` при обнаружении новых моделей

### Requirement: Конфигурация обнаружения
Система ДОЛЖНА поддерживать конфигурацию обнаружения для каждого провайдера.

#### Scenario: Конфигурация статического обнаружения
- **WHEN** провайдер имеет `discovery: {type: "static", models: [...]}` в конфигурации
- **THEN** используется `StaticDiscovery` с настроенными моделями

#### Scenario: Конфигурация динамического обнаружения (future)
- **WHEN** провайдер имеет `discovery: {type: "dynamic", endpoint: "..."}` в конфигурации
- **THEN** используется динамическое обнаружение (когда реализовано)
