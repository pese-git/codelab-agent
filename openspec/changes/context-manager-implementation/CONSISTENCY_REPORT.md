## Исправленные проблемы

### ✅ Критические (исправлены)

#### 1. Противоречие в описании логирования кэша
**Файл:** `specs/context-storage/spec.md:10`  
**Было:** "логирует debug `file_cache_miss` при первом, без лога при попадании"  
**Стало:** "логирует debug `file_cache_miss` при промахе кэша, без лога при попадании (hit)"  
**Статус:** ✅ Исправлено

#### 2. Неполная сигнатура метода в спецификации
**Файл:** `specs/single-strategy/spec.md:10,28`  
**Было:** `ensure_context_fits(envelope, max_context_tokens, reserved_tokens)`  
**Стало:** `ensure_context_fits(envelope, *, max_context_tokens, reserved_tokens)`  
**Статус:** ✅ Исправлено

#### 3. Неопределённое поле в модели данных
**Файл:** `specs/agent-context-models/spec.md:88`  
**Было:** Поле `last_accessed` отсутствовало в описании  
**Стало:** Добавлено описание: `last_accessed: float = 0.0` — временная метка последнего доступа (для LRU eviction)  
**Статус:** ✅ Исправлено

---

### ✅ Средние (исправлены)

#### 4. Нестабильная терминология "Phase" vs "Фаза"
**Файлы:** `proposal.md`, `design.md`, `tasks.md`, `specs/*`  
**Было:** Смешанное использование "Phase 0-6" и "Фаза 0-6"  
**Стало:** Унифицировано на "Фаза 0-6" во всех документах  
**Статус:** ✅ Исправлено

#### 5. Нестабильные пути конфигурации feature flags
**Файлы:** `proposal.md`, `design.md`, `tasks.md`  
**Было:** Короткие пути (`gather.enabled`, `storage.enabled`)  
**Стало:** Полные пути (`agents.context.gather.enabled`, `agents.context.storage.enabled`)  
**Статус:** ✅ Исправлено

#### 6. Нестабильная терминология "hydration" vs "гидрация"
**Файл:** `specs/context-multiagent/spec.md:119,123`  
**Было:** "hydration vs epoch", "используется hydration или epoch"  
**Стало:** "гидрация vs эпоха", "используется гидрация или эпоха"  
**Статус:** ✅ Исправлено
